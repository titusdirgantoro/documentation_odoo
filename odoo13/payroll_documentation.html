🏗 1. Ringkasan Arsitektur

Ekosistem payroll Odoo 13 custom ini dibangun dengan arsitektur API-driven: Odoo bukan menghitung payroll secara lokal, melainkan mengirim master data (employee, contract, company, government regulation) ke backend payroll API eksternal (Node.js), lalu memetakan response kembali ke objek hr.payslip Odoo.


Seluruh implementasi aktif bertumpu pada jalur fp_* (fp_employee → fp_hr_payroll), bukan pada modul lama if_hr_payroll atau if_payroll.

🔄 14. Alur Payroll End-to-End

1

Setup Master Data

fp_employee membentuk master employee, contract, salary component, BPJS, NPWP, cutoff. fp_hr_payroll menyinkronkan ke API payroll (Node.js).

2

Attendance & Overtime Processing

if_attendance mencatat check-in/out dengan approval flow. if_overtime menambah overtime request & rate bertingkat. Attendance approved otomatis membuat record overtime.

3

Pre-Payroll Validation

if_hr_payroll_attendance memastikan tidak ada attendance/overtime/leave yang belum approved. Data dikonversi menjadi actual_working_days, overtime amount, dan absence deduction.

4

Run Payroll (API Call)

fp_hr_payroll mengirim payload ke backend API. Response dipetakan ke hr.payslip: basic salary, allowances, deductions, BPJS, PPh21, THR, pesangon.

5

Loan Injection

if_loan menambah cicilan pinjaman jatuh tempo sebagai deduction, mengurangi THP. Rollback otomatis jika payslip dihapus.

6

Backpay Processing

if_hr_payroll_attendance memproses backpay untuk kontrak terlambat atau perubahan komponen gaji.

7

Batch Close & Journal

if_hr_payroll_journal membentuk account.move berdasarkan mapping komponen → akun. Posting dipisah berdasarkan costing_type (direct vs indirect).

8

Finalize & Lock

Payslip di-finalize ke backend. Attendance/overtime/leave/contract dikunci. if_employee_exit memaksa pelunasan loan karyawan resign.

9

Distribution

PDF payslip dibuat, dikirim ke mobile app dan/atau broadcast via WhatsApp.

⚠️ 16. Catatan Penting & Hidden Coupling

🚨 Ketergantungan API Eksternal
Arsitektur sangat bergantung ke payroll API (Node.js). Perubahan requirement harus dicek di: Odoo module, kontrak payload, cutoff logic, DAN endpoint backend.
⚠️ Hidden Coupling — Lock Lintas Model
if_hr_payroll_attendance memakai lock payroll lintas model dan monkey patch pada hr.leave. Perubahan kecil di attendance/leave bisa berdampak langsung ke payroll.
⚠️ Loan Settlement on Exit
if_employee_exit memaksa pelunasan sisa loan saat finalisasi payslip resign. Pastikan saldo loan sudah benar sebelum finalize.
💡 Concurrent Batch
fp_hr_payroll mencegah double-processing: batch hanya bisa di-run jika batch tipe sama sudah finalized.
💡 Backpay
Diproses oleh if_hr_payroll_attendance untuk kontrak terlambat, perubahan basic salary, dan perubahan komponen. Memperhitungkan missed days dan cutoff.
