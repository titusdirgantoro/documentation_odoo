Section 4
Sequence: Upload Data Pipeline
Urutan lengkap dari klik "Upload" sampai AI Summary tersedia untuk user.
Upload → Chunk → AI Summary Sequence
sequenceDiagram actor User participant UI as Next.js UI participant API as Backend API participant SBStorage as Supabase Storage participant DB as Supabase DB participant BG as Background Job participant Claude as Claude API participant EMB as Embedding Model User->>UI: Drag & drop file (.xlsx/.csv) UI->>UI: Validasi: ukuran, format, dept tag UI->>API: POST /upload (file + dept_tag + org_id) API->>SBStorage: Simpan file SBStorage-->>API: file_url API->>DB: INSERT uploads (status: pending) DB-->>API: upload_id API-->>UI: 202 Accepted (processing) UI-->>User: ✅ "Upload diterima, sedang dianalisis..." BG->>DB: Poll: SELECT uploads WHERE status='pending' DB-->>BG: upload_id BG->>DB: UPDATE uploads SET status='processing' BG->>SBStorage: Ambil file SBStorage-->>BG: raw file bytes loop Setiap sheet / grup logis / date range BG->>Claude: Extract struktur + identifikasi metadata Claude-->>BG: structured content + metadata JSON BG->>EMB: Generate embedding (1536 dims) EMB-->>BG: embedding vector BG->>DB: INSERT chunks (content, embedding, metadata, period) end BG->>DB: UPDATE uploads SET status='done', chunk_count=N DB-->>UI: Realtime subscription trigger UI-->>User: 🎉 "Analisis selesai! AI Summary tersedia"
⚠️ Jika Gagal: status → failed, error_msg terisi, klien melihat: "Upload gagal — [alasan]. Harap unggah ulang."
1

Client Upload

Drag-and-drop .xlsx, .xls, .csv max 50MB (Basic) / 200MB (Standard+). Tag departemen wajib diisi.

2

Simpan & Catat

File disimpan ke Supabase Storage. Row baru dibuat di tabel uploads dengan status pending.

3

Background Processing

Job mengambil file, memecahnya berdasarkan sheet + grup data + date range. Bukan satu file = satu chunk.

4

Claude Extraction

Setiap grup dikirim ke Claude untuk ekstrak struktur, kategori, periode, dan metrik kunci.

5

Embedding & Simpan

Konten + metadata disimpan ke tabel chunks bersama embedding vektor 1536 dimensi via text-embedding-3-small.

6

Notifikasi User

Status upload diperbarui ke done. UI mendapat sinyal realtime → user diberitahu untuk melihat AI Summary.

Sequence: RAG Query (Ask AI)
Bagaimana pertanyaan user diubah menjadi jawaban AI yang grounded pada data klien — bukan halusinasi.
RAG (Retrieval-Augmented Generation) Query Flow
sequenceDiagram actor User as User (PRO) participant UI as Web UI participant API as Backend API participant EMB as Embedding Model participant DB as Supabase (pgvector) participant Claude as Claude API User->>UI: Ketik pertanyaan: "Kenapa revenue Oktober turun?" UI->>API: POST /ask (question, org_id, session_id) API->>EMB: Embed question → vector (1536 dims) EMB-->>API: query_vector Note over API,DB: Filter WAJIB: org_id DULU, baru vector search API->>DB: SELECT chunks WHERE org_id=:id
ORDER BY embedding <=> query_vec LIMIT 10 DB-->>API: Top 10 chunk paling relevan Note over API,Claude: Inject chunk sebagai konteks, bukan web search API->>Claude: [System prompt] + [10 chunks] + [User question] Claude-->>API: Jawaban terstruktur + sumber yang dipakai API->>DB: INSERT messages (role,content,sources JSONB) API-->>UI: Response + source traceability UI-->>User: Jawaban AI dengan referensi chunk sumber Note over User: Jika pertanyaan tidak bisa dijawab dari data,
Claude menyatakannya secara eksplisit
🔍 Retrieval Filter Order (Selalu dalam urutan ini):
1. WHERE org_id = :org_id — isolasi, tidak boleh dilewati
2. AND period_start >= :start (jika filter waktu diberikan)
3. AND period_end <= :end (jika filter waktu diberikan)
4. AND metadata->>'category' = :cat (jika kategori diberikan)
5. ORDER BY embedding <=> :query_vec — kemiripan semantik paling terakhir

🚨 ISOLASI

Jangan pernah query tanpa org_id filter terlebih dahulu. RLS adalah cadangan, bukan penjaga utama.

🚫 TANPA ZAPIER

Hanya integrasi API langsung. Tidak ada alat otomatisasi middleware. Full code ownership.

✋ VALIDASI DULU

David wajib menguji setiap saran tugas AI sendiri sebelum diserahkan ke tim. Jangan skip langkah ini.

🔀 BACKEND TERPISAH

ImpactFirst (Node+PostgreSQL) dan ImpactLab (Next.js+Supabase) adalah dua stack berbeda. Jangan digabung.

📊 SELALU DATA-DRIVEN

Setiap respons AI harus didasarkan pada data yang diunggah klien. Bukan murni percakapan/halusinasi.

🧪 MVP DULU

Y1 = eksperimen. Zero marketing push. Buktikan transformasi di perusahaan David sendiri sebelum scaling.

📐 CEK SKALA

Sebelum ship fitur apa pun, tanya: "Apakah fitur ini masih berfungsi saat ImpactLab memiliki 100+ tenants dan menjadi >50% revenue?" Jika tidak jelas, revisi dulu.
