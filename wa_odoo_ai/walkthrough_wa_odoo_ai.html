Python (FastAPI + LangChain) · PostgreSQL · Meta WA Cloud API
Dokumen Arsitektur — Maret 2026

1 Ringkasan Sistem

Sistem ini memungkinkan Manajer mendapatkan laporan data ERP (penjualan, stok, gudang, dll.) cukup dengan mengirim pesan WhatsApp ke Bot. AI akan menerjemahkan pertanyaan tersebut menjadi SQL query, mengeksekusinya ke database Odoo, lalu mengirim hasilnya kembali dalam format yang mudah dibaca.

Penjelasan Per Fase

🔴 Fase 1 — Keamanan (Validasi Identitas)

Middleware menerima pesan dari Meta API, lalu langsung mengecek nomor pengirim. Jika nomor tidak ada di daftar whitelist manajer, proses berhenti total — AI tidak dipanggil, database tidak disentuh. Ini menghemat resource dan mencegah akses ilegal.

🔵 Fase 2 — Translasi Bahasa Manusia ke SQL

Middleware mengirimkan pertanyaan manajer beserta skema database (daftar kolom dari View yang sudah disederhanakan) ke AI. AI menganalisis konteks dan menghasilkan SQL query yang tepat. Contoh: SELECT SUM(total_penjualan) FROM view_ai_sales_report WHERE tanggal = CURRENT_DATE

🟢 Fase 3 — Eksekusi Data (Read-Only)

Middleware mengeksekusi SQL buatan AI ke PostgreSQL. Koneksi menggunakan akun ai_reader yang hanya bisa SELECT. Sekalipun AI membuat perintah DELETE atau DROP, PostgreSQL akan menolaknya.

🟠 Fase 4 — Formatting Hasil

Data mentah dari database (misal: [(15050000,)]) dikirim kembali ke AI untuk diubah menjadi kalimat yang ramah, profesional, dan mudah dibaca di layar HP WhatsApp.

🚨 PERINGATAN: Tanpa lapis ke-4 (Database Views), AI akan memiliki akses ke seluruh 800+ tabel Odoo termasuk res_users yang menyimpan hash password. Membuat Views khusus adalah WAJIB.

6 Kekurangan & Risiko

#KekuranganDampakMitigasi
1 Computed Fields Odoo tidak terlihat Field yang dihitung oleh Python ORM (bukan disimpan di DB) tidak bisa di-query Buat logika kalkulasi di dalam SQL View
2 Ketergantungan Cloud Jika Meta WA API atau OpenAI/Gemini down, bot tidak berfungsi Siapkan fallback notifikasi / antrian pesan
3 Risiko Halusinasi AI (±1-2%) AI bisa salah menginterpretasi pertanyaan ambigu Tambahkan disclaimer di balasan + validasi query
4 Biaya API LLM Setiap pertanyaan memakan token berbayar (~Rp 200-1000/pertanyaan) Monitor usage, batasi jumlah pertanyaan/hari
5 Maintenance View Jika struktur tabel Odoo berubah (upgrade modul), View harus diperbarui Dokumentasikan View dan jadwalkan review berkala
